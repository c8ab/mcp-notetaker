== Note

The primary entity. Represents a single atomic note captured during a working session.

[cols="1,1,1,2", options="header"]
|===
| Field | Type | Constraints | Notes

| note_id
| TEXT
| PRIMARY KEY
| UUID v7, generated by the application (ADR-0004)

| session_id
| TEXT
| NOT NULL
| Associates the note with a working session. Immutable after creation.

| user_id
| TEXT
| NOT NULL
| The person or identity taking the note. Immutable after creation.

| agent
| TEXT
| NOT NULL
| The AI agent that produced the note. Immutable after creation.

| title
| TEXT
| (nullable)
| Short summary. Mutable.

| content
| TEXT
| NOT NULL
| Note body in AsciiDoc format. Mutable.

| parent_note_id
| TEXT
| REFERENCES notes(note_id), (nullable)
| Self-referential FK for note threading. Mutable.

| context_url
| TEXT
| (nullable)
| URL or file path the note relates to. Mutable.

| tags
| TEXT
| (nullable)
| JSON array of strings, e.g. `["architecture", "decision"]`. Mutable.

| created_at
| TEXT
| NOT NULL
| ISO 8601 timestamp. Set once at creation. Immutable.

| updated_at
| TEXT
| NOT NULL
| ISO 8601 timestamp. Updated on every mutation.

| deleted_at
| TEXT
| (nullable)
| ISO 8601 timestamp. NULL = active, non-NULL = soft-deleted (ADR-0003).
|===

== FTS5 Virtual Table

[cols="1,2", options="header"]
|===
| Property | Value

| Table name
| notes_fts

| Content columns
| title, content

| Tokenizer
| unicode61 (default FTS5 tokenizer; handles case folding)

| Content source
| Manually synced from the notes table at the application level
|===

The FTS5 table is an external content table. Rows are inserted, updated (delete + re-insert), and deleted in sync with the notes table by the NoteRepository. Soft-deleted notes have their FTS rows removed to ensure they never appear in search results.

== SQL Schema

[source,sql]
----
CREATE TABLE IF NOT EXISTS notes (
  note_id        TEXT PRIMARY KEY,
  session_id     TEXT NOT NULL,
  user_id        TEXT NOT NULL,
  agent          TEXT NOT NULL,
  title          TEXT,
  content        TEXT NOT NULL,
  parent_note_id TEXT REFERENCES notes(note_id),
  context_url    TEXT,
  tags           TEXT,  -- JSON array
  created_at     TEXT NOT NULL,
  updated_at     TEXT NOT NULL,
  deleted_at     TEXT
);

CREATE INDEX IF NOT EXISTS idx_notes_session_id ON notes(session_id);
CREATE INDEX IF NOT EXISTS idx_notes_user_id ON notes(user_id);
CREATE INDEX IF NOT EXISTS idx_notes_agent ON notes(agent);
CREATE INDEX IF NOT EXISTS idx_notes_parent_note_id ON notes(parent_note_id);
CREATE INDEX IF NOT EXISTS idx_notes_created_at ON notes(created_at);
CREATE INDEX IF NOT EXISTS idx_notes_deleted_at ON notes(deleted_at);

CREATE VIRTUAL TABLE IF NOT EXISTS notes_fts USING fts5(
  title,
  content,
  content='',
  contentless_delete=1
);
----

A note may reference one parent note. A parent note may have zero or more child notes. This enables note threading and hierarchical organization.
